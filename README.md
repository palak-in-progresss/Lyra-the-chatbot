# Lyra - Conversational AI Learning Assistant

Lyra is a professional, portfolio-grade conversational AI assistant built using **Python, Streamlit, Supabase, and Google Gemini 2.5 Flash**. Designed specifically for students and junior developers, Lyra integrates a consistent personality with multiple specialized assistance modes, secure user authentication, and persistent cloud-backed conversation history.

---

## 🌟 Features

### 1. Secure Authentication & Session Persistence
Allows users to register and log in securely using Supabase Auth. It bypasses iframe security blocks (common in cloud deployments like Streamlit Community Cloud) using a secure cookie-bridge to restore user sessions automatically on page refreshes.

### 2. Multi-Session Conversation History
Users can create separate conversation threads, review historical sessions from the sidebar, or delete old sessions. Each session is dynamically auto-titled based on the user's first prompt.

### 3. Domain-Specific Assistant Modes
Using a sidebar selector, users can switch the active expertise mode. Each mode triggers tailored system instructions:
* **🤖 General Assistant:** General concepts, brainstorming, and creative tasks.
* **🎓 Study Buddy:** Summarization, active recall quizzes, and interactive learning.
* **💻 DSA Helper:** Step-by-step algorithms and data structures guidance. Lyra uses hint-based support rather than giving away code directly, and provides Big-O time/space complexity analysis.
* **💼 Career Mentor:** Guidance on tech paths, project ideas, portfolio building, and interview preparation.
* **📄 Resume Reviewer:** PDF resume text extraction and detailed evaluation (Strengths, Weaknesses, Actionable suggestions, ATS keywords, and Missing skills).
* **🌌 Space Mentor:** Astronomy, astrophysics, and stargazing recommendations. Includes custom generated quizzes.

### 4. Interactive Resume Conversations
Upload a PDF in the Resume Reviewer mode. The text is parsed using PyPDF, evaluated by the LLM, and output directly into the chat log, allowing you to ask follow-up questions about the critique (e.g., *"How can I rephrase my project to include the missing skills you mentioned?"*).

### 5. PostgreSQL Row-Level Security (RLS)
The database backend is fully secured. Row-Level Security policies ensure that users can only read, write, or delete their own chat records, isolating session details completely.

## 🛠️ Tech Stack

* **Frontend:** Streamlit
* **Database & Auth:** Supabase (PostgreSQL & Auth Services)
* **AI Model:** Google Gemini 2.5 Flash (via the modern `google-genai` SDK)
* **Session Persistence:** `streamlit-cookies-controller` (cookie-level token exchange)
* **PDF Parser:** PyPDF (robust text extraction from byte streams)
* **Environment Configuration:** python-dotenv
* **Styling:** Custom CSS (Vanilla CSS for glassmorphic dark theme)

## 🚀 Installation & Setup

Follow these steps to run Lyra locally:

### 1. Prerequisites
Ensure you have **Python 3.9+** and a **Supabase Project** set up.

### 2. Install Dependencies
Open your terminal in the project root directory and run:
```bash
pip install -r requirements.txt
```

### 3. Database Setup (Supabase)
In your Supabase SQL Editor, run the following script to initialize the tables and setup RLS policies:
```sql
-- Create Users Table
create table users (
    id uuid primary key default gen_random_uuid(),
    created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Create Sessions Table
create table sessions (
    id uuid primary key default gen_random_uuid(),
    user_id uuid references users(id) on delete cascade not null,
    title text not null,
    created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Create Conversations Table
create table conversations (
    id bigint generated by default as identity primary key,
    user_id uuid references users(id) on delete cascade not null,
    session_id uuid references sessions(id) on delete cascade not null,
    role text not null check (role in ('user', 'assistant')),
    message text not null,
    mode text not null,
    created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- Enable RLS
alter table public.users enable row level security;
alter table public.sessions enable row level security;
alter table public.conversations enable row level security;

-- Create Policies (Restrict access to the logged-in owner)
create policy "Allow users to insert their own profile" on public.users for insert with check (auth.uid() = id);
create policy "Allow users to read their own profile" on public.users for select using (auth.uid() = id);
create policy "Allow users to manage their own sessions" on public.sessions for all using (auth.uid() = user_id) with check (auth.uid() = user_id);
create policy "Allow users to manage their own conversations" on public.conversations for all using (auth.uid() = user_id) with check (auth.uid() = user_id);
```

### 4. Configure Your Environment
Create a `.env` file in the root directory:
```env
GEMINI_API_KEY=your_actual_gemini_key_here
SUPABASE_URL=your_supabase_project_url_here
SUPABASE_KEY=your_supabase_anon_public_key_here
```
*(Get your Gemini key from [Google AI Studio](https://aistudio.google.com/) and Supabase credentials from your Supabase Dashboard settings).*

### 5. Launch the Application
Start the Streamlit local server:
```bash
streamlit run app.py
```
A browser tab will automatically open at `http://localhost:8501`.
